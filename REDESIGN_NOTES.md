# Mobile Redesign Notes (2026)

## Что сохранено из исходного флоу
- Вход по телефону с обязательным согласием.
- Альтернативный вход через провайдеры (Yandex ID / MTS ID).
- Подтверждение через SMS-код.
- Подтверждение паролем после кода.

## Что улучшено в UX
- Чёткая иерархия: hero-блок, затем ключевое действие, затем альтернативы.
- Один главный CTA на экране и вторичные действия визуально отделены.
- Больше воздуха, улучшенная читаемость, единая сетка и отступы.
- Мобильный-first контейнер с поддержкой узких экранов.
- Red accent сохранён как основа брендинга и фокуса.

## Тренды 2026, но без потери usability
- Мягкие градиенты и глубина слоёв вместо тяжёлого декоративного стекла.
- Выразительная типографика Manrope с сильной иерархией веса.
- Контрастные акценты действий и аккуратные motion-переходы.

## Edge cases в прототипе
- Неверный формат номера.
- Отсутствие согласия с условиями.
- Неверный SMS-код.
- Истёкший код (демо: `000000`).
- Повторная отправка кода по таймеру.
- Короткий пароль.
- Временная блокировка аккаунта (демо: `blocked123`).
- Экран помощи и вспомогательные сценарии.

## Демо-сценарий happy path
1. Ввести номер и отметить чекбокс.
2. Нажать "Получить код".
3. Ввести код `123456`.
4. Ввести пароль длиной 8+ символов.
5. Нажать "Войти".
